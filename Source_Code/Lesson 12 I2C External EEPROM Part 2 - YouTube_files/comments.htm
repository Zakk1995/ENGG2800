+Callum Fleming hey there, sorry for the delay in reply.  However, you asked a pretty involved question.  It may spawn a video :)  I will try to explain here but it may not be very clear, there is only so much you can do with text...lol.

Ok, first of all looking at the 2464.c code provided by CCS C it appears it was coded for only one chip so you will have to modify the 2464.c file functions so that the read an write function will have a third parameter that will take in the chip select bits.

Looking at the data sheet for a 24XX64 chip starting with section 2.0 it tells us that the chip has an input select set of bits.  A0, A1, and A2.  This is three bits so it lets us know you can stack up to 8 chips on the i2c buss (i.e. 0x00 to 0x07)

These chip select pins have to be set hardware wise.  For example if you want somthing to have the addres of 0x05 -> 0b00000101 then you would have to connect pin A0 to Vcc, A1 to GND, and A2 to Vcc on the chip you want to have address 0x05 and so on for any additional chips.

Once the hardware is set up you then need to move to the code.

In the data sheet section 5.0 it explains the address control byte formatting.  This is effectivly a 10 bit chunk that is put on the i2c buss.  The first bit is the start bit followed by the control code, which for this chip is "1010".  Then next comes the address section formatted |A2|A1|A0|.  This is where you will have to modify the code.  The section I am referring to is:

void write_ext_eeprom(long int address, BYTE data)
{
   short int status;
   i2c_start();
   i2c_write(0xa0);
   i2c_write((address>>8)&0x1f);
   i2c_write(address);
   i2c_write(data);
   i2c_stop();
   i2c_start();
   status=i2c_write(0xa0);
   while(status==1)
   {
      i2c_start();
      status=i2c_write(0xa0);
   }
   i2c_stop();
}

BYTE read_ext_eeprom(long int address) {
   BYTE data;
   i2c_start();
   i2c_write(0xa0);
   i2c_write((address>>8)&0x1f);
   i2c_write(address);
   i2c_start();
   i2c_write(0xa1);
   data=i2c_read(0);
   i2c_stop();
   return(data);
}

It will need to be re-coded with another parameter that "OR's" the address in.  Like below:


void write_ext_eeprom(long int address, BYTE data, int chip)
{
   short int status;
   chip<<=1;
   i2c_start();
   i2c_write(0xa0 | chip);
   i2c_write((address>>8)&0x1f);
   i2c_write(address);
   i2c_write(data);
   i2c_stop();
   i2c_start();
   status=i2c_write(0xa0);
   while(status==1)
   {
      i2c_start();
      status=i2c_write(0xa0);
   }
   i2c_stop();
}

BYTE read_ext_eeprom(long int address,int chip) {
   BYTE data;
   chip<<=1;
   i2c_start();
   i2c_write(0xa0 | chip);
   i2c_write((address>>8)&0x1f);
   i2c_write(address);
   i2c_start();
   i2c_write(0xa1);
   data=i2c_read(0);
   i2c_stop();
   return(data);
}


Now I did this realy fast and have not tested it but the thought behind it is that you will enter a number 0-7 for the chip you want in the "chip" parameter. This will then be in bianary 0b0111.  Since if we OR'd this in now we would have the following: 10100111 the "111" is one bit off since it needs to be to the left one bit, referring to the datasheet section 5.0.  Thus, the piece of code "chip<<=1;" this will store "1110" in the chip variable instead of "0111" that way when it is OR'd with the 0xa0 it will come out, in bianary, to "10101110"  This way the chip will select chip 7 and not chip zero.

Take care and good luck.﻿
